# Scoring Framework — Startup Radar

## 1. Objectif du scoring

Le scoring ne doit pas prétendre donner une vérité absolue. Il sert à :

- comparer des startups entre elles ;
- prioriser les analyses ;
- détecter les opportunités ;
- détecter les risques ;
- justifier une décision `Pass`, `Watch`, `Deep Dive`, `Investable`.

Le score doit rester explicable et modifiable manuellement.

---

## 2. Score global

Score total sur 100 points.

| Critère | Poids |
|---|---:|
| Market Score | 15 |
| Problem Pain Score | 10 |
| Product Maturity Score | 10 |
| Traction Score | 15 |
| Team Score | 15 |
| Technical Moat Score | 15 |
| Valuation Score | 10 |
| Investor Quality Score | 5 |
| Exit Potential Score | 5 |
| **Total** | **100** |

---

## 3. Kamel Edge Score

Score séparé sur 25 points. Il mesure l'avantage personnel d'analyse.

| Question | Points |
|---|---:|
| Je comprends techniquement le produit | 5 |
| Je peux tester/reproduire une partie du produit | 5 |
| J'ai accès à des experts ou utilisateurs pour valider | 5 |
| La startup touche mes domaines stratégiques | 5 |
| Je peux apporter un avantage : marché, contact, expertise | 5 |
| **Total** | **25** |

Interprétation :

- 0-8 : pas d'avantage personnel ;
- 9-15 : avantage moyen ;
- 16-20 : avantage fort ;
- 21-25 : avantage très fort.

---

## 4. Décision recommandée

| Score global | Kamel Edge | Décision |
|---:|---:|---|
| < 50 | quelconque | Pass |
| 50-65 | < 15 | Watch |
| 50-65 | >= 15 | Deep Dive |
| 65-80 | >= 15 | Strong Watch |
| 80+ | >= 18 | Investigation prioritaire |
| 80+ | >= 21 + valorisation acceptable | Investable potentiel |

---

## 5. Red Flags

Une startup doit être rejetée ou fortement pénalisée si :

- elle se présente comme IA sans technologie propre ;
- elle dépend d'une seule API sans moat ;
- elle n'a aucun client ou pilote après 18 mois ;
- la valorisation est trop haute comparée à la traction ;
- l'équipe manque de CTO ou de compétence métier ;
- les fondateurs ne maîtrisent pas le problème ;
- le produit est impossible à tester ;
- le marché est trop petit ;
- les droits IP ou data sont flous ;
- le business model dépend exclusivement de subventions ;
- le cycle de vente est trop long sans financement suffisant.

---

## 6. Méthode de valorisation simplifiée

### Early Stage sans revenus

Utiliser :

- Scorecard Method ;
- comparables de levées ;
- qualité équipe ;
- taille marché ;
- maturité produit ;
- traction non financière.

### SaaS avec revenus

Utiliser :

- ARR ;
- croissance mensuelle ;
- marge brute ;
- churn ;
- CAC/LTV si disponible ;
- multiples ARR comparables.

### Deeptech / biotech / hardware

Utiliser :

- maturité scientifique ;
- brevets ;
- validation terrain/labo ;
- partenaires académiques ;
- intensité capitalistique ;
- jalons réglementaires ;
- dilution future probable.

---

## 7. Scénarios de rendement

Chaque startup doit être évaluée en scénarios :

| Scénario | Description | Multiple |
|---|---|---:|
| Échec | perte totale ou quasi totale | x0 |
| Survie | faible croissance, pas d'exit majeur | x1-x2 |
| Bon succès | acquisition ou forte croissance | x5-x10 |
| Grand gagnant | leader de catégorie | x20-x100 |

Le radar doit produire une espérance qualitative, pas une promesse de rendement.

---

## 8. Sorties de scoring

Pour chaque startup :

- score global ;
- Kamel Edge ;
- score risque ;
- red flags ;
- décision ;
- justification courte ;
- prochaines questions de due diligence.
