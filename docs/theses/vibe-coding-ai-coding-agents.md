# Thesis: Vibe Coding / AI Coding Agents

## Definition

Vibe Coding refers to software creation through natural language prompts and iterative AI-assisted development. The category includes AI coding IDEs, autonomous software engineering agents, app builders, frontend generators, codebase-understanding tools and open-source coding assistants.

## Why it matters

This segment can compress the software creation cycle from weeks or months to hours or days. It can also expand software creation to non-developers, product managers, designers, founders and small businesses.

## Key metrics to track

### Growth metrics

- ARR / revenue latest.
- ARR growth month-over-month and year-over-year.
- Active users and weekly active users.
- Paid conversion rate.
- Enterprise customers.
- Net revenue retention.
- Seat expansion.

### Product metrics

- Tasks completed per user.
- Pull requests created or merged.
- Acceptance rate of generated code.
- Test pass rate.
- Bug rate and rollback rate.
- Time saved per task.
- Codebase context window and repository understanding quality.

### Risk metrics

- Dependency on external foundation models.
- Gross margin after inference costs.
- Security incidents or leaked secrets.
- Poor generated code quality.
- User backlash on pricing or rate limits.
- Enterprise compliance gaps.
- Platform risk from OpenAI, Anthropic, Microsoft and Google.

## Investment logic

The best companies in the category combine:

1. Strong product-led growth.
2. High developer retention.
3. Enterprise adoption.
4. Deep codebase understanding.
5. Workflow integration with GitHub, CI/CD, cloud and IDEs.
6. Strong pricing power.
7. Low enough inference cost to sustain gross margins.

## Red flags

- Pure wrapper with no workflow moat.
- High usage but weak paid conversion.
- Very high valuation without verified ARR.
- Heavy model API dependency with weak margins.
- Security failures in generated applications.
- Users switching quickly between tools.
- No enterprise controls.

## Upcoming events to monitor

- Major model update.
- Agentic workflow release.
- Enterprise pricing launch.
- Security/compliance certification.
- Strategic partnership or acquisition rumor.
- Late-stage round or secondary transaction.
- IPO preparation.

## Radar interpretation

This segment should have its own watchlist because product usage, funding and technical capability can change every few weeks.

Short-term 1/2/3/6-month momentum forecasts are useful here, but they must be interpreted as momentum scenarios rather than fundamental predictions.
