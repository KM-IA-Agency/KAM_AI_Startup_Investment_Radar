# AI Tools Trending by Category — July 2026

## Purpose

This document turns a broad market list into a usable radar taxonomy.

The objective is not only to know which tools are trendy, but to classify them by:

- role in the builder workflow;
- investability;
- strategic relevance;
- public/private exposure;
- core versus adjacent relevance to Vibe Coding.

## 1. General AI Assistants

Key tools:

- ChatGPT
- Gemini
- Claude
- DeepSeek
- Qwen
- Kimi
- Mistral
- GLM / GLM-5.2
- Grok

### Radar interpretation

These tools form the top-level AI interface. They are not all vibe-coding tools, but they shape how users research, design, code, automate and analyze.

### Strategic note

GLM-5.2 from Z.ai is a specific signal to monitor in July 2026 because developer adoption appears to be rising due to coding/agent capability and low cost.

## 2. AI Search

Key tools:

- Perplexity
- ChatGPT Search
- Gemini / AI Mode Google
- You.com
- Genspark
- Liner

### Radar interpretation

AI search is strategically important because it changes the user behavior from searching pages to asking for structured answers.

This can disrupt classical search, SEO, content discovery and research workflows.

## 3. Productivity / Office / Enterprise

Key tools:

- Microsoft 365 Copilot
- Google Gemini Workspace
- Notion AI
- Grammarly
- NotebookLM
- Glean
- Gamma

### Radar interpretation

This category is highly relevant to enterprise adoption and AI distribution. The boundary between native AI tool and classical software with AI is disappearing.

For investment analysis, track:

- enterprise distribution;
- seat expansion;
- retention;
- workflow depth;
- integration with documents, email, meetings and internal search.

## 4. Code / Software Development

Key tools:

- GitHub Copilot
- Cursor
- Claude Code
- OpenAI Codex
- Windsurf
- Replit Agent
- Lovable
- Bolt.new
- v0
- Base44
- Devin
- Z.ai GLM-5.2 / ZCode

### Radar interpretation

This is probably the most explosive category in 2026.

The key reason: AI is moving from text generation to product construction.

Metrics to track:

- ARR;
- developer adoption;
- PRs generated or merged;
- code acceptance rate;
- agent task success rate;
- deployment reliability;
- enterprise adoption;
- inference cost and gross margin;
- security incidents.

## Builder workflow model

The complete builder stack is:

```text
idea
→ research
→ knowledge synthesis
→ prototype
→ code generation
→ deployment
→ workflow automation
→ design assets
→ video/audio/demo
→ presentation
→ sales
→ monitoring
```

No single tool owns the whole stack yet.

## Investment lens

For the radar, classify tools into four groups:

1. **Core investment target**: standalone high-growth company with direct value capture.
2. **Public comparable**: listed company with exposure through a product line.
3. **Strategic platform risk**: large model/platform that can crush or acquire startups.
4. **Workflow utility**: useful for builders but not necessarily investable.

## Priority zones for Kamel

Given the user's projects, the most useful categories are:

1. Code / software development.
2. Agents and automation.
3. AI search / research.
4. Productivity and knowledge tools.
5. Design/video/audio for GTM and demos.

These categories can directly accelerate the creation of SaaS MVPs, dashboards, AI agents, data platforms and investor reports.
